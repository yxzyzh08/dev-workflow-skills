# Security Auditor Boundary Examples

Use these examples before taking ownership of a borderline review request.

- Use `security-auditor`: the current code change, architecture decision, or dependency update needs a scoped security review.
- Do not use `security-auditor`: the request is a general functional review with no security focus.
- Do not use `security-auditor`: the human wants an unbounded full-repo audit but has not explicitly asked for that broader scope.
- Do not use `security-auditor`: the next step is implementing a fix rather than auditing the risk.

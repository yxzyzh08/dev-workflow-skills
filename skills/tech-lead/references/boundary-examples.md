# Tech Lead Boundary Examples

Use these examples before taking ownership of borderline requests.

- Use `tech-lead`: architecture is stable and the next step is to define scope, task slicing, interfaces, and data models.
- Do not use `tech-lead`: the system shape itself is still unsettled; return to `system-architect`.
- Do not use `tech-lead`: the request is to implement code or fix a code-level bug; use `developer`.
- Do not use `tech-lead`: the request is to author or revise E2E plans and suites; use `test-engineer`.

# Developer Boundary Examples

Use these examples when a request is close to another skill boundary.

- Use `developer`: scope and design are ready, and the next step is test-first implementation or a code-level fix.
- Do not use `developer`: `detail.md` is missing key contract information; route back to `tech-lead`.
- Do not use `developer`: Delivery QA has not finished failure classification yet; wait for the routed bug analysis instead of guessing.
- Do not use `developer`: the problem is an architecture gap rather than an implementation defect; route to `system-architect`.
